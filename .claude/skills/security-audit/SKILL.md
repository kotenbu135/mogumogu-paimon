# Security Audit スキル

セキュリティ脆弱性を体系的にスキャンし、重要度付きのレポートを出力します。OWASP Top 10 をベースに、このプロジェクト固有のリスク（ファイルアップロード・JSON解析・静的エクスポート）も対象とします。

## 実行フロー

2フェーズに分けて精度を高めます。各フェーズを `Agent` ツール（`context: fork`）で実行してメインコンテキストを節約します。

**重要**: フェーズ1の結果をメインエージェントで受け取り、フェーズ2のプロンプトに直接埋め込んで渡すこと。

---

## フェーズ 1: ソースコード収集・チェックポイント抽出

Agent ツール（`context: fork`）で以下を実行し、結果を受け取る。

```
以下のファイルを読み込み、セキュリティ監査のチェックポイントを抽出してください。

対象ファイル（優先度順に読み込む）:
- webapp/src/app/ 以下の全 page.tsx / layout.tsx
- webapp/src/components/ 以下の全コンポーネント
- webapp/src/lib/ 以下の全ロジックファイル
- webapp/next.config.ts（セキュリティヘッダー設定）
- webapp/src/app/api/ 以下（APIルートがあれば）

各ファイルから以下のチェックポイントを抽出してください:

1. **入力検証**: ユーザー入力（フォーム・ファイル・URLパラメータ）の検証処理
2. **ファイルアップロード**: MIMEタイプ・サイズ・内容の検証
3. **JSON解析**: 外部JSONのスキーマ検証・型チェック
4. **XSS リスク**: dangerouslySetInnerHTML・ユーザー入力の直接レンダリング
5. **セキュリティヘッダー**: CSP・X-Frame-Options・HSTS 等の設定
6. **計算量DoS**: 無制限ループ・再帰・データ処理
7. **画像・リソースパス**: ユーザー入力を src/href に直接使用している箇所
8. **依存関係**: package.json の脆弱バージョン（既知 CVE の有無）

出力形式: カテゴリごとに「ファイルパス:行番号 — 観察内容」の形式で列挙する。
```

---

## フェーズ 2: リスク評価・レポート出力

フェーズ1の出力を受け取り、Agent ツール（`context: fork`）で以下を実行する。`{チェックポイント}` にはフェーズ1の出力をそのまま貼り付けること。

```
以下のチェックポイントをもとにセキュリティリスクを評価し、監査レポートを作成してください。

チェックポイント:
{フェーズ1の出力}

重要度の定義:
- 🔴 Critical: 即時悪用可能（XSS・任意コード実行・機密データ漏洩）
- 🟠 High: 条件付きで悪用可能（CSRF・認証バイパス・DoS）
- 🟡 Medium: 多層防御の欠如（バリデーション不足・セキュリティヘッダー未設定）
- 🟢 Low: ベストプラクティスからの逸脱（軽微な情報漏洩・不要な詳細エラー）
- ✅ OK: 適切に対処済み

評価ルール:
- ユーザー入力が無検証でDOMに挿入される → 🔴
- ファイルアップロードでMIMEタイプ・サイズ未検証 → 🟠
- JSONスキーマ検証なしで外部データを処理 → 🟠
- CSP・X-Frame-Options 未設定 → 🟡
- 無制限の計算量（上限なしループ） → 🟠
- 画像srcにユーザー入力を直接使用 → 🟠

出力形式（Markdownテーブル）:

## Security Audit レポート

### 入力検証

| ファイル:行 | 問題の概要 | 重要度 | 推奨対応 |
|------------|-----------|--------|---------|
| ...        | ...       | 🔴/🟠/🟡/🟢/✅ | ... |

### ファイルアップロード
（同形式）

### JSON解析・スキーマ検証
（同形式）

### XSSリスク
（同形式）

### セキュリティヘッダー
（同形式）

### 計算量DoS
（同形式）

### サマリー

| 重要度 | 件数 |
|--------|------|
| 🔴 Critical | N |
| 🟠 High | N |
| 🟡 Medium | N |
| 🟢 Low | N |
| ✅ OK | N |
| 合計 | N |

### 優先対応リスト

重要度順に修正推奨事項を3件以内でリストアップする。

問題が発見されなかったカテゴリは省略すること。
```

---

## 注意事項

- 発見した脆弱性は自動修正せず、レポート出力のみ行うこと（修正は `/tdd-workflow` と組み合わせて対応）
- Critical・High は即時対応を推奨、Medium 以下はスプリント内で検討
- 静的解析のみのため、ランタイム挙動は対象外
