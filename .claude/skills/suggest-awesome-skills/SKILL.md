# Awesome GitHub Copilotスキルの提案

現在のリポジトリコンテキストを分析し、[GitHub awesome-copilotリポジトリ](https://github.com/github/awesome-copilot/blob/main/docs/README.skills.md)からこのリポジトリに未導入の関連エージェントスキルを提案します。エージェントスキルは[awesome-copilotリポジトリのskillsフォルダ](https://github.com/github/awesome-copilot/tree/main/skills)に自己完結したフォルダとして存在し、各フォルダには指示と任意の同梱アセットを含む`SKILL.md`ファイルが入っています。

## 手順

1. **利用可能なスキルの取得**: [awesome-copilot README.skills.md](https://github.com/github/awesome-copilot/blob/main/docs/README.skills.md)からスキル一覧と説明を抽出する。WebFetchツールを使用すること。
2. **ローカルスキルのスキャン**: `.claude/skills/`フォルダ内の既存スキルフォルダを探索する
3. **説明の抽出**: ローカルの`SKILL.md`ファイルのフロントマターから`name`と`description`を読み取る
4. **リモートバージョンの取得**: 各ローカルスキルについて、rawのGitHub URL（例: `https://raw.githubusercontent.com/github/awesome-copilot/main/skills/<スキル名>/SKILL.md`）を使ってawesome-copilotリポジトリから対応する`SKILL.md`をWebFetchで取得する
5. **バージョンの比較**: ローカルスキルのコンテンツとリモートバージョンを比較し、以下を特定する:
   - 最新のスキル（完全一致）
   - 古いスキル（コンテンツに差異あり）
   - 古いスキルの主要な差異（説明、指示、同梱アセット）
6. **コンテキストの分析**: チャット履歴、リポジトリファイル、現在のプロジェクトニーズをレビューする
7. **既存スキルとの照合**: このリポジトリに既に存在するスキルと照合する
8. **関連性のマッチング**: 利用可能なスキルと特定したパターンおよび要件を比較する
9. **選択肢の提示**: 関連するスキルを説明・根拠・可用性ステータス（古いスキルを含む）と共に表示する
10. **検証**: 提案するスキルが既存スキルでカバーされていない価値を追加できることを確認する
11. **出力**: 提案・説明・awesome-copilotスキルへのリンクおよび類似ローカルスキルへのリンクを含む構造化テーブルを提供する
    **待機** — ユーザーが特定のスキルのインストールまたは更新を指示するまで待つこと。**指示なしにインストール・更新を実行してはならない。**
12. **アセットのダウンロード/更新**: 要求されたスキルについて、以下を自動的に実行する:
    - 新しいスキルを`.claude/skills/`フォルダにフォルダ構造を維持してダウンロードする
    - 古いスキルをawesome-copilotの最新バージョンで置き換えて更新する
    - `SKILL.md`と同梱アセット（スクリプト、テンプレート、データファイル）の両方をダウンロードする
    - ファイルの内容を調整しないこと
    - WebFetchツールでアセットをダウンロードし、全コンテンツの取得が確実な場合はBashツールで`curl`を使用してもよい
    - TodoWriteツールで進捗を管理する

## コンテキスト分析基準

**リポジトリパターン**:
- 使用されているプログラミング言語（.cs、.js、.py、.ts など）
- フレームワークの指標（ASP.NET、React、Azure、Next.js など）
- プロジェクトの種類（Webアプリ、API、ライブラリ、ツール、インフラ）
- 開発ワークフロー要件（テスト、CI/CD、デプロイ）
- インフラとクラウドプロバイダー（Azure、AWS、GCP）

**チャット履歴コンテキスト**:
- 最近の議論と課題
- 機能リクエストや実装ニーズ
- コードレビューのパターン
- 開発ワークフロー要件
- 専門タスクのニーズ（ダイアグラム作成、評価、デプロイ）

## ローカルスキル探索プロセス

1. `.claude/skills/`ディレクトリ内のすべてのフォルダを一覧表示する
2. 各フォルダの`SKILL.md`フロントマターを読み取り、`name`と`description`を抽出する
3. 各スキルフォルダ内の同梱アセットを一覧表示する
4. 既存スキルとその機能の包括的なインベントリを構築する
5. このインベントリを使用して重複の提案を避ける

## バージョン比較プロセス

1. 各ローカルスキルフォルダについて、リモートの`SKILL.md`を取得するためのrawのGitHub URLを構築する:
   - パターン: `https://raw.githubusercontent.com/github/awesome-copilot/main/skills/<スキル名>/SKILL.md`
2. WebFetchツールでリモートバージョンを取得する
3. ファイル全体のコンテンツ（フロントマターと本文を含む）を比較する
4. 具体的な差異を特定する:
   - **フロントマターの変更**（name、description）
   - **指示の更新**（ガイドライン、例、ベストプラクティス）
   - **同梱アセットの変更**（新規追加、削除、変更されたアセット）
5. 古いスキルの主要な差異を文書化する
6. 更新が必要かどうかを判断するために類似度を計算する

## スキル構造要件

エージェントスキルの仕様に基づき、各スキルは以下を含むフォルダとして構成される:
- **`SKILL.md`**: フロントマター（`name`、`description`）と詳細な指示を含むメインの指示ファイル
- **任意の同梱アセット**: `SKILL.md`から参照されるスクリプト、テンプレート、参照データ、その他のファイル
- **フォルダの命名**: ハイフン区切りの小文字（例: `azure-deployment-preflight`）
- **名前の一致**: `SKILL.md`フロントマターの`name`フィールドはフォルダ名と一致しなければならない

## フロントマター構造

awesome-copilotのスキルは`SKILL.md`で以下のフロントマター形式を使用する:
```markdown
---
name: 'skill-name'
description: 'このスキルが提供する内容と使用タイミングの簡単な説明'
---
```

## 要件

- WebFetchツールを使ってawesome-copilotリポジトリのスキルドキュメントからコンテンツを取得する
- 個々のスキルコンテンツのダウンロードにはWebFetchツールまたはBashツール（gh CLI）を使用する
- Globツールを使って`.claude/skills/`ディレクトリ内の既存スキルをスキャンする
- Readツールでローカルの`SKILL.md`ファイルのYAMLフロントマターを読み取り、名前と説明を抽出する
- ローカルスキルとリモートバージョンを比較して古いスキルを検出する
- 重複を避けるためにリポジトリの既存スキルと比較する
- 現在のスキルライブラリのカバレッジのギャップに焦点を当てる
- 提案するスキルがリポジトリの目的と技術スタックに合致することを検証する
- 各提案に明確な根拠を提供する
- awesome-copilotスキルと類似ローカルスキルの両方へのリンクを含める
- 古いスキルを明確に特定し、具体的な差異を記載する
- 同梱アセットの要件と互換性を考慮する
- テーブルと分析以外の追加情報やコンテキストは提供しない

## アイコン凡例

- ✅ インストール済みかつ最新
- ⚠️ インストール済みだが古い（更新あり）
- ❌ リポジトリに未導入

## 更新処理

古いスキルが特定された場合:
1. ⚠️ステータスで出力テーブルに含める
2. 「提案理由」列に具体的な差異を記載する
3. 主要な変更点を示した更新推奨を提供する
4. ユーザーが更新を要求した場合、ローカルスキルフォルダ全体をリモートバージョンで置き換える
5. `.claude/skills/`ディレクトリ内のフォルダ位置を維持する
6. 更新された`SKILL.md`と一緒にすべての同梱アセットをダウンロードする
