# DRY Check スキル

コードベース全体を走査して重複定義・重複ロジックを検出し、統合方針を優先度付きで提案します。DRY（Don't Repeat Yourself）原則に基づき、保守性向上を目的とします。

## 実行フロー

3フェーズで実行します。各フェーズを `Agent` ツール（`context: fork`）で実行してメインコンテキストを節約します。

---

## フェーズ 1: 定数・型定義の重複検出

Agent ツール（`context: fork`）で以下を実行し、結果を受け取る。

```
以下のファイルを読み込み、定数・型定義の重複を検出してください。

対象ファイル:
- webapp/src/lib/constants.ts
- webapp/src/lib/types.ts
- webapp/src/app/ 以下の全 page.tsx
- webapp/src/components/ 以下の全コンポーネント

検出対象:
1. **定数の重複定義**: 同じ値・同じ意味の定数が複数ファイルで定義されている
   例: `const MAX_SIZE = 5` が2箇所以上に存在
2. **型定義の重複**: 同一または類似の型・インターフェースが複数箇所で定義されている
3. **配列・オブジェクトリテラルの重複**: 同じラベル/値ペアのリストが複数箇所に存在
4. **ハードコードされた文字列**: 同じ文字列リテラルが3箇所以上に出現

出力形式:
各重複について:
- 重複の種類（定数/型/配列/文字列）
- 出現箇所（ファイルパス:行番号 のリスト）
- 内容の概要（値または型の要約）
```

## フェーズ 2: ロジック・パターンの重複検出

Agent ツール（`context: fork`）で以下を実行し、結果を受け取る。

```
以下のファイルを読み込み、ロジック・コードパターンの重複を検出してください。

対象ファイル:
- webapp/src/lib/ 以下の全ファイル
- webapp/src/hooks/ 以下の全ファイル
- webapp/src/components/ 以下の全ファイル
- webapp/src/app/ 以下の全 page.tsx

検出対象:
1. **同一計算ロジック**: 同じアルゴリズム・計算式が複数箇所で実装されている
2. **同一データ変換**: 同じマッピング・フィルタ・マージ処理が複数箇所に存在
   例: `{ ...a, ...b }` の同じパターンが複数コンポーネントで使われている
3. **同一条件分岐**: 同じ if/switch パターンが複数ファイルで繰り返されている
4. **コンポーネントの重複**: 類似した JSX 構造が複数コンポーネントで使われている

出力形式:
各重複について:
- 重複の種類
- 出現箇所（ファイルパス:行番号 のリスト）
- 重複内容の要約（コードの抜粋）
```

---

## フェーズ 3: 統合提案レポート出力

フェーズ1・2の出力を受け取り、Agent ツール（`context: fork`）で以下を実行する。`{フェーズ1の出力}` `{フェーズ2の出力}` を貼り付けること。

```
以下の重複検出結果をもとに、DRY Check レポートを作成してください。

フェーズ1の結果（定数・型定義）:
{フェーズ1の出力}

フェーズ2の結果（ロジック・パターン）:
{フェーズ2の出力}

重要度の定義:
- 🔴 High: 3箇所以上で重複 / ビジネスロジックの重複（バグが伝播するリスクあり）
- 🟡 Medium: 2箇所で重複 / 定数・型定義の重複（変更漏れのリスクあり）
- 🟢 Low: 軽微な重複（将来的に重複が増えるリスクあり）

出力形式（Markdownテーブル）:

## DRY Check レポート

### 定数・型定義の重複

| 重複内容 | 出現箇所 | 重要度 | 統合方針 |
|---------|---------|--------|---------|
| ...     | ...     | 🔴/🟡/🟢 | ... |

### ロジック・パターンの重複

| 重複内容 | 出現箇所 | 重要度 | 統合方針 |
|---------|---------|--------|---------|
| ...     | ...     | 🔴/🟡/🟢 | ... |

### サマリー

| 重要度 | 件数 |
|--------|------|
| 🔴 High | N |
| 🟡 Medium | N |
| 🟢 Low | N |
| 合計 | N |

### 優先対応リスト

重要度順に統合推奨事項を上位3件リストアップし、統合先ファイルと方針を具体的に記載する。

統合方針の例:
- 「`constants.ts` に移動し、各ファイルからインポートする」
- 「カスタムフック `useXxx` に抽出する」
- 「共通ユーティリティ関数 `calcXxx` に抽出して `lib/utils.ts` に配置する」

重複が発見されない場合は「DRY違反は検出されませんでした ✅」と出力すること。
```

---

## 注意事項

- 検出のみ行い、自動修正はしないこと（修正は `/tdd-workflow` で対応）
- 意図的な重複（テストフィクスチャ・ドキュメント例など）は除外すること
- 修正優先度の目安: 🔴 → 即時対応、🟡 → スプリント内で検討、🟢 → バックログへ
