# アーキテクチャブループリント生成

コードベースを体系的に分析し、詳細なアーキテクチャドキュメントを生成します。

## 概要

このスキルは以下を自動検出・ドキュメント化します：

- **技術スタック**: 言語、フレームワーク、ライブラリ
- **アーキテクチャパターン**: レイヤード、マイクロサービス、Clean Architecture など
- **コンポーネント関係図**: 視覚化（C4、UML、フロー図）
- **実装パターン**: 設計パターン、ベストプラクティス

## 主要な検出項目

### 1. 技術スタック認識

- バックエンド: Node.js、.NET、Java、Python など
- フロントエンド: React、Vue、Angular、Svelte など
- データベース: PostgreSQL、MongoDB、SQL Server など
- インフラ: Docker、Kubernetes、AWS、Azure など

このリポジトリの例：
```
- フロントエンド: Next.js 16 + React 19 + TypeScript 5
- スタイリング: Tailwind CSS 4
- テスト: Vitest 4
- デプロイ: GitHub Pages (静的サイト)
```

### 2. アーキテクチャパターン認識

- **静的サイト**: GitHub Pages デプロイで単純
- **フロント/バック分離**: 該当なし（SPA）
- **レイヤリング**: Component → Hook → Lib → Utils

### 3. ドキュメント生成対象（15-17 セクション）

1. **概要**: プロジェクト目的、技術スタック
2. **アーキテクチャ図**: C4 モデル、コンポーネント図
3. **ディレクトリ構造**: フォルダ役割、命名規則
4. **コンポーネント設計**: React コンポーネント階層
5. **ルーティング**: Next.js App Router 構成
6. **状態管理**: グローバルステート管理方法
7. **データフロー**: ユーザー入力 → 処理 → 表示
8. **スタイリング戦略**: Tailwind CSS 設定、カスタムテーマ
9. **テスト戦略**: Vitest テスト配置、カバレッジ
10. **パフォーマンス**: 最適化手法、バンドルサイズ
11. **CI/CD**: GitHub Pages デプロイパイプライン
12. **セキュリティ**: XSS、CSRF 対策
13. **アクセシビリティ**: WCAG 準拠確認
14. **スケーリング計画**: 機能追加時の拡張性
15. **依存関係**: npm パッケージ、セキュリティアップデート

## 使用場面

- 新規開発者のオンボーディング
- アーキテクチャレビュー
- 技術的意思決定の記録
- 機能追加時の設計ガイド
- 長期的なアーキテクチャ保守

## 生成物フォーマット

```markdown
# [プロジェクト名] - アーキテクチャブループリント

## 概要
...

## 技術スタック
...

## アーキテクチャ図
[C4 / UML / フロー図]

## ディレクトリ構造
...

## コンポーネント設計
...
```

## ベストプラクティス

- 定期的に更新（機能追加時）
- チーム全体で共有
- コード例を含める
- 意思決定の根拠を記載
