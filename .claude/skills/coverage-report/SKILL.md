# Coverage Report スキル

Vitest カバレッジを実行し、未テスト箇所を優先度付きでリストアップして次に書くべきテストを提案します。

## 手順

### ステップ 1: カバレッジ設定を確認する

`webapp/vitest.config.ts` を読み込み、`coverage` 設定の有無を確認する。

設定がない場合は以下を追記するよう提案する:

```ts
coverage: {
  provider: 'v8',
  reporter: ['text', 'json', 'html'],
  include: ['src/**/*.{ts,tsx}'],
  exclude: ['src/**/*.test.{ts,tsx}', 'src/**/*.d.ts'],
},
```

### ステップ 2: カバレッジを実行する

Agent ツール（`context: fork`）を使って以下を実行する:

```
以下のコマンドを実行してカバレッジ結果を取得してください。

cd webapp && npm test -- --coverage --reporter=json 2>&1

出力から以下を抽出してください:
- 全体のカバレッジ率（行・ブランチ・関数）
- カバレッジ 0% の関数リスト（ファイルパスと関数名）
- カバレッジが低い（50% 未満）ファイルのリスト
```

### ステップ 3: 未テスト箇所を分析する

カバレッジ結果を基に `webapp/src/lib/` の対象ファイルを読み込み、未テスト関数の内容を把握する。

分析観点:
- スコア計算・再構築成功率などのロジック関数を優先
- UI コンポーネントよりもビジネスロジックを優先
- 既存テスト（`webapp/src/lib/__tests__/`）を参照して重複を避ける

### ステップ 4: テスト提案を出力する

以下の形式で未テスト箇所と提案を出力する:

```markdown
## カバレッジレポート

### 全体カバレッジ
| 種別 | カバレッジ率 |
|------|------------|
| 行   | XX%        |
| ブランチ | XX%    |
| 関数 | XX%        |

### 未テスト関数（優先度順）
| 優先度 | ファイル | 関数名 | 推奨テストケース |
|--------|----------|--------|-----------------|
| 高     | src/lib/xxx.ts | funcName | 正常系・エッジケース |

### 最小テストケース案
各未テスト関数について最小限のテストコードを提示する。
```

### ステップ 5: tdd-workflow への接続を促す

テスト追加が必要な場合は `/tdd-workflow` スキルを使って TDD サイクルで実装するよう案内する。

## 注意事項

- カバレッジ計測には `@vitest/coverage-v8` パッケージが必要（未インストールの場合は `npm install -D @vitest/coverage-v8` を提案する）
- テスト配置先: `webapp/src/lib/__tests__/`
- 実行前に `cd webapp && npm test` が通ることを確認する
