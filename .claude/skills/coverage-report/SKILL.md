# カバレッジレポートスキル

Vitest のカバレッジ計測を実行し、未テスト箇所を優先度付きでリストアップして「最小テストケース案」を提示します。

## 実行手順

以下の 3 フェーズを順番に実行すること。

---

## フェーズ 1: カバレッジ計測

1. `webapp/vitest.config.ts` の coverage 設定を確認する
   - `coverage.provider` が設定されていない場合のみ、以下を `vitest.config.ts` に追加提案する:
     ```ts
     coverage: {
       provider: 'v8',
       reporter: ['text', 'json', 'html'],
       include: ['src/**/*.{ts,tsx}'],
       exclude: ['src/**/*.test.{ts,tsx}', 'src/**/*.d.ts', 'src/app/**'],
     }
     ```

2. カバレッジ付きでテストを実行する:
   ```bash
   cd webapp && npm test -- --coverage 2>&1
   ```
   - `@vitest/coverage-v8` が未インストールでエラーになる場合:
     ```bash
     cd webapp && npm install -D @vitest/coverage-v8
     ```

3. `webapp/coverage/coverage-summary.json` を Read ツールで読み込み、以下を出力する:
   - **全体サマリー**: 行・ブランチ・関数・ステートメントのカバレッジ率
   - **カバレッジ 0% のファイル一覧**
   - **カバレッジ 50% 未満のファイル一覧**

---

## フェーズ 2: 未テスト箇所の優先度付け

フェーズ 1 で取得したカバレッジ結果を元に、以下を実行する:

1. カバレッジ 0% または 50% 未満のファイルを Read ツールで読み込む

2. エクスポートされている関数・クラスメソッドを列挙し、以下の基準で優先度を付ける:
   - **High**: ビジネスロジック・スコア計算・データ変換など中核ロジック
   - **Medium**: ユーティリティ関数・バリデーション・フォーマット処理
   - **Low**: UI ヘルパー・定数定義・型定義のみ

3. 以下の形式で優先度付きテーブルを出力する:
   | 優先度 | ファイル | 関数名 | 未カバー行数 | 理由 |
   |--------|----------|--------|-------------|------|
   | High   | ...      | ...    | ...         | ...  |

---

## フェーズ 3: 最小テストケース案の提示

フェーズ 2 の優先度付きリストを元に、以下のルールでテストケース案を作成する:

- **High 優先度が 5 件以上**の場合: High のみ対象
- **High 優先度が 5 件未満**の場合: High + Medium を対象

各対象関数について:

1. 対象関数のソースを Read ツールで読み込み、引数・戻り値・条件分岐を把握する

2. 以下の形式で「最小テストケース案」を提示する:

   ### `関数名` (ファイルパス)

   **役割**: （1行で説明）

   **テストケース案**:
   ```ts
   // 正常系
   it('should ...', () => {
     // Arrange
     // Act
     // Assert
   })

   // エッジケース
   it('should handle ... when ...', () => {
     // ...
   })
   ```

3. 全対象関数のテストケース案を提示した後、以下のメッセージを出力する:
   > テストを実装する場合は `/tdd-workflow` を実行してください。
   > RED フェーズで上記テストケース案を貼り付けることで TDD サイクルを開始できます。

---

## 依存・前提条件

- `webapp/vitest.config.ts` に `coverage` 設定が必要（なければフェーズ 1 で追加提案）
- `@vitest/coverage-v8` パッケージが必要（未インストールならフェーズ 1 で案内）

## 関連スキル

- `/tdd-workflow` — テストケース案を実装する際の TDD サイクルスキル
