# カバレッジレポートスキル

Vitest のカバレッジ計測を実行し、未テスト箇所を優先度付きでリストアップして「最小テストケース案」を提示します。

## 実行方法

以下の3フェーズを順番に `Agent` ツール（`context: fork`）で実行すること。

---

## フェーズ 1: カバレッジ計測

Agent ツールで以下のプロンプトを実行する（`context: fork`）:

```
カバレッジ計測フェーズを実行してください。

1. vitest.config.ts の coverage 設定を確認する
   - ファイル: webapp/vitest.config.ts
   - `coverage.provider` が設定されていることを確認する
   - 設定がなければ以下を追加提案する:
     ```ts
     coverage: {
       provider: 'v8',
       reporter: ['text', 'json', 'html'],
       include: ['src/**/*.{ts,tsx}'],
       exclude: ['src/**/*.test.{ts,tsx}', 'src/**/*.d.ts', 'src/app/**'],
     }
     ```

2. カバレッジ付きでテストを実行する
   ```bash
   cd webapp && npm test -- --coverage --reporter=json 2>&1
   ```
   - エラーが出た場合は `@vitest/coverage-v8` の追加インストールを提案する:
     `cd webapp && npm install -D @vitest/coverage-v8`

3. カバレッジ結果を収集する
   - `webapp/coverage/coverage-summary.json` を読み込む
   - ファイルごとの行・ブランチ・関数カバレッジを抽出する

完了後、以下を報告してください:
- 全体カバレッジのサマリー（行・ブランチ・関数・ステートメント）
- カバレッジ 0% のファイル一覧
- カバレッジ 50% 未満のファイル一覧
```

---

## フェーズ 2: 未テスト箇所の優先度付けリスト化

フェーズ 1 の結果を受け取り、Agent ツールで以下を実行する（`context: fork`）:

```
未テスト箇所の優先度付けを実行してください。

カバレッジサマリー: {{フェーズ1のカバレッジ結果}}

1. カバレッジ 0% の関数を特定する
   - 対象ファイルを Read ツールで読み込む
   - エクスポートされている関数・クラスメソッドを列挙する
   - 関数の役割（ビジネスロジック / ユーティリティ / UI ヘルパー）を分類する

2. 優先度を付ける（以下の基準で High / Medium / Low を判定）
   - **High**: ビジネスロジック・スコア計算・データ変換など中核ロジック
   - **Medium**: ユーティリティ関数・バリデーション・フォーマット処理
   - **Low**: UI ヘルパー・定数定義・型定義のみ

3. 優先度付きテーブルを作成する（以下の形式）:
   | 優先度 | ファイル | 関数名 | 未カバー行数 | 理由 |
   |--------|----------|--------|-------------|------|
   | High   | ...      | ...    | ...         | ...  |

完了後、優先度付きテーブルを報告してください。
```

---

## フェーズ 3: 最小テストケース案の提示

フェーズ 2 の結果（優先度付きリスト）を受け取り、Agent ツールで以下を実行する（`context: fork`）:

```
最小テストケース案を提示してください。

優先度リスト: {{フェーズ2の優先度付きテーブル}}

High 優先度の関数について、以下の手順でテストケース案を作成する:

1. 対象関数のソースを読み込む
   - 引数・戻り値の型を確認する
   - 条件分岐・エッジケースを把握する

2. 各関数に対して「最小テストケース案」を提示する（以下の形式）:

   ### `関数名` (ファイルパス)

   **役割**: （1行で説明）

   **テストケース案**:
   ```ts
   // 正常系
   it('should ...', () => {
     // Arrange
     // Act
     // Assert
   })

   // エッジケース
   it('should handle ... when ...', () => {
     // ...
   })
   ```

3. tdd-workflow スキルへの接続を促す
   - 以下のメッセージを出力する:
     > テストを実装する場合は `/tdd-workflow` を実行してください。
     > RED フェーズで上記テストケース案を貼り付けることで TDD サイクルを開始できます。

完了後、High 優先度の全関数についてテストケース案を報告してください。
```

---

## 依存・前提条件

- `webapp/vitest.config.ts` に `coverage` 設定が必要（なければフェーズ1で追加提案）
- `@vitest/coverage-v8` パッケージが必要:
  ```bash
  cd webapp && npm install -D @vitest/coverage-v8
  ```

## 関連スキル

- `/tdd-workflow` — テストケース案を実装する際の TDD サイクルスキル
