# Spec Sync スキル

`docs/spec/` の仕様ファイルと `webapp/src/lib/` の実装コードを比較し、乖離点を分類してMarkdownテーブル形式でレポートを出力します。

## 実行方法

以下の2フェーズを順番に `Agent` ツール（`context: fork`）で実行すること。

---

## フェーズ 1: 仕様ファイルを読み込んでチェックポイントを抽出する

Agent ツールで以下のプロンプトを実行する（`context: fork`）:

```
docs/spec/ の全仕様ファイルを読み込み、検証すべきチェックポイントを抽出してください。

対象ファイル:
- docs/spec/artifact.md       → 聖遺物の種別・スロット・メインステータス定義
- docs/spec/enhancement.md    → 強化ロール計算の定義・境界値
- docs/spec/substat-rolls.md  → サブステロール数計算ロジック・定数
- docs/spec/good-format.md    → GOOD フォーマット仕様・フィールド定義

各ファイルから以下を抽出してください:
1. 計算式（数式・アルゴリズム）
2. 定数・閾値・境界値（数値リスト、テーブル）
3. 入力/出力の型・フィールド名
4. 特殊ケース・例外処理の記述

出力形式:
仕様ファイルごとに箇条書きでチェックポイントを列挙すること。
```

---

## フェーズ 2: 実装コードと照合して乖離レポートを出力する

フェーズ 1 の結果（チェックポイント一覧）を受け取り、Agent ツールで以下を実行する（`context: fork`）:

```
以下のチェックポイントと実装コードを照合し、乖離レポートを作成してください。

チェックポイント:
{{フェーズ1のチェックポイント一覧}}

照合対象の実装ファイル:
- webapp/src/lib/scoring.ts          → スコア計算・ロール数ロジック
- webapp/src/lib/reconstruction.ts   → 再構築成功率計算
- webapp/src/lib/constants.ts        → 定数・閾値定義
- webapp/src/lib/types.ts            → 型定義・フィールド名
- webapp/src/lib/__tests__/          → テストカバレッジ確認

照合手順:
1. 各チェックポイントを対応する実装ファイルで検索する
2. 仕様の計算式・定数と実装を比較する
3. テストファイルで該当ロジックがカバーされているか確認する
4. 以下の3分類で乖離点を記録する:
   - 仕様と異なる: 実装が仕様と異なる値・ロジックを持つ
   - 仕様未実装: 仕様に定義されているが実装が存在しない
   - テスト未カバー: 実装はあるがテストが存在しない・不十分

出力形式（Markdownテーブル）:

## Spec Sync レポート

### artifact.md

| 仕様箇所 | 実装ファイル | 実装状況 | 分類 | 対応アクション |
|---------|------------|---------|------|--------------|
| （仕様の記述） | （ファイル:行番号） | （実装の状態） | 仕様と異なる / 仕様未実装 / テスト未カバー / OK | （推奨アクション） |

### enhancement.md
（同形式）

### substat-rolls.md
（同形式）

### good-format.md
（同形式）

### サマリー

| 分類 | 件数 |
|------|------|
| 仕様と異なる | N |
| 仕様未実装 | N |
| テスト未カバー | N |
| OK | N |
| 合計 | N |

乖離が0件の場合は「仕様と実装は一致しています」と出力すること。
```

---

## 利用シーン

- 仕様変更後に実装への反映漏れを検出したい場合
- 新機能実装前に既存の仕様カバレッジを把握したい場合
- コードレビュー前に仕様準拠を確認したい場合

## 注意事項

- 仕様ドキュメントの「TODO」「未定義」箇所は照合対象外とすること
- 実装上の最適化（計算順序の変更など）は仕様との乖離とみなさないこと
- 乖離を発見した場合は自動修正せず、レポート出力のみ行うこと
- 修正が必要な場合は `/tdd-workflow` スキルを使用すること
