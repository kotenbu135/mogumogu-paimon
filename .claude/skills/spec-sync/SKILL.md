# Spec Sync スキル

`docs/spec/` の仕様ファイルと実装コードを比較し、乖離点を重要度付きで分類してMarkdownテーブル形式でレポートを出力します。仕様ドリフトの早期発見と品質可視化が目的です。

## 実行フロー

2フェーズに分けるのは、仕様の読み込みと実装照合を分離して精度を高めるためです。各フェーズを `Agent` ツール（`context: fork`）で実行することでメインコンテキストを節約できます。

**重要**: フェーズ1の結果をメインエージェントで受け取り、フェーズ2のプロンプトに直接埋め込んで渡すこと。

---

## フェーズ 1: チェックポイント抽出

Agent ツール（`context: fork`）で以下を実行し、結果を受け取る。

```
docs/spec/ の全仕様ファイルを読み込み、実装と照合すべきチェックポイントを抽出してください。

対象ファイル:
- docs/spec/artifact.md       → 聖遺物の種別・スロット・メインステータス定義
- docs/spec/enhancement.md    → 強化ロール計算の定義・境界値
- docs/spec/substat-rolls.md  → サブステロール数計算ロジック・定数
- docs/spec/good-format.md    → GOOD フォーマット仕様・フィールド定義

各ファイルから以下を抽出してください:
1. 計算式（数式・アルゴリズム）
2. 定数・閾値・境界値（数値リスト、テーブル）
3. 入力/出力の型・フィールド名
4. 特殊ケース・例外処理の記述

「TODO」「未定義」と記載されている箇所は対象外とすること。

出力形式: 仕様ファイルごとに箇条書きでチェックポイントを列挙する。
```

---

## フェーズ 2: 実装照合・レポート出力

フェーズ1の出力を受け取り、Agent ツール（`context: fork`）で以下を実行する。`{チェックポイント}` にはフェーズ1の出力をそのまま貼り付けること。

```
以下のチェックポイントと実装コードを照合し、乖離レポートを作成してください。

チェックポイント:
{フェーズ1の出力}

照合対象の実装ファイル:
- webapp/src/lib/scoring.ts          → スコア計算・ロール数ロジック
- webapp/src/lib/reconstruction.ts   → 再構築成功率計算
- webapp/src/lib/constants.ts        → 定数・閾値定義
- webapp/src/lib/types.ts            → 型定義・フィールド名
- webapp/src/lib/__tests__/          → テストカバレッジ確認

照合手順:
1. 各チェックポイントを対応する実装ファイルで検索する
2. 仕様の計算式・定数と実装を比較する
3. テストファイルで該当ロジックがカバーされているか確認する
4. 以下の重要度で乖離点を分類する:
   - 🔴 仕様と異なる: 実装が仕様と異なる値・ロジックを持つ（即時修正推奨）
   - 🟡 仕様未実装: 仕様に定義されているが実装が存在しない（対応検討）
   - 🟡 テスト未カバー: 実装はあるがテストが不十分（対応検討）
   - ✅ OK: 仕様と実装が一致している

実装上の最適化（計算順序の変更など）は乖離とみなさないこと。

出力形式（Markdownテーブル）:

## Spec Sync レポート

### artifact.md

| 仕様箇所 | 実装ファイル | 実装状況 | 重要度 | 対応アクション |
|---------|------------|---------|--------|--------------|
| （仕様の記述） | （ファイル:行番号） | （実装の状態） | 🔴/🟡/✅ | （推奨アクション） |

### enhancement.md
（同形式）

### substat-rolls.md
（同形式）

### good-format.md
（同形式）

### サマリー

| 重要度 | 件数 |
|--------|------|
| 🔴 仕様と異なる | N |
| 🟡 仕様未実装 | N |
| 🟡 テスト未カバー | N |
| ✅ OK | N |
| 合計 | N |

乖離が0件の場合は「仕様と実装は一致しています ✅」と出力すること。
```

---

## 注意事項

- 乖離を発見した場合は自動修正せず、レポート出力のみ行うこと（修正は `/tdd-workflow` で対応）
- 修正優先度の目安: 🔴 → 即時対応、🟡 → スプリント内で検討
