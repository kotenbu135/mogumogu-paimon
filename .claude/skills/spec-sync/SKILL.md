# Spec Sync スキル

`docs/spec/` の仕様ファイルと `webapp/src/lib/` の実装コードを比較し、乖離点を検出してレポートを出力します。

## 手順

### ステップ 1: 仕様ファイルを読み込む

Agent ツール（`context: fork`）を使って以下の仕様ファイルを読み込む:

- `docs/spec/artifact.md` — 聖遺物仕様
- `docs/spec/enhancement.md` — 強化ロール計算仕様
- `docs/spec/substat-rolls.md` — サブステロール数計算ロジック
- `docs/spec/good-format.md` — GOOD フォーマット仕様

各ファイルから以下を抽出する:
- 計算式・定数・閾値
- 境界値（最小・最大・特殊値）
- データ型・フォーマット仕様

### ステップ 2: 対応する実装ファイルを特定する

`webapp/src/lib/` 配下のファイルを確認し、各仕様に対応する実装ファイルを特定する。

対応関係の例:
- `artifact.md` → 聖遺物の型定義・バリデーション
- `enhancement.md` → 強化ロール計算ロジック
- `substat-rolls.md` → サブステロール数計算
- `good-format.md` → GOOD フォーマットのパース・変換

### ステップ 3: 仕様と実装を比較する

各仕様項目について実装コードを照合し、以下の観点で乖離を検出する:

1. **計算式の一致**: 仕様の数式が実装に正確に反映されているか
2. **定数・閾値の一致**: 仕様に記載された数値と実装の定数が一致するか
3. **境界値の処理**: エッジケース（0、最大値等）が仕様通りに処理されるか
4. **未実装項目**: 仕様に定義されているが実装がない機能
5. **テストカバレッジ**: `webapp/src/lib/__tests__/` に対応するテストがあるか

### ステップ 4: 差分レポートを出力する

以下の形式で乖離点をレポートする:

```markdown
## 仕様-実装整合チェックレポート

### サマリー
- チェック対象仕様ファイル: X 件
- 仕様と異なる実装: X 件
- 仕様未実装: X 件
- テスト未カバー: X 件

### 詳細

| 仕様箇所 | 仕様内容 | 実装状況 | 分類 | 対応アクション |
|----------|----------|----------|------|----------------|
| artifact.md:L12 | ... | ... | 仕様と異なる | 実装を修正する |
| enhancement.md:L34 | ... | 未実装 | 仕様未実装 | 実装を追加する |
| substat-rolls.md:L8 | ... | 実装あり | テスト未カバー | テストを追加する |

### 推奨アクション
優先度順に修正・追加すべき項目を列挙する。
```

分類の定義:
- **仕様と異なる**: 実装は存在するが仕様と食い違っている
- **仕様未実装**: 仕様に定義されているが実装がない
- **テスト未カバー**: 実装は正しいがテストで検証されていない

## 注意事項

- 仕様ファイルのパスは `docs/spec/` から確認すること
- 実装ファイルは `webapp/src/lib/` を基点に探索する
- 乖離が見つかった場合は `/tdd-workflow` スキルで修正テストを先行して書くことを推奨する
